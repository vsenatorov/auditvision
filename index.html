// capabilities

Everything you need to
audit your cluster

From raw API events to compliance reports — Audit Radar covers the full audit lifecycle.

⚡
Live Event Stream
Real-time feed of all Kubernetes API audit events. Filter by actor, namespace, verb, resource, risk level, result code. Stream updates every second without page refresh.
🧠
AI Risk Scoring
Every event scored HIGH / MEDIUM / LOW by IBM Granite 3.2 running locally via Ollama. No data leaves your cluster. Strategic choice for Red Hat environments.
🔔
Alert Rules
Configurable webhook alerts to Slack and email. Built-in rules for HIGH risk events and human DELETE actions. Custom trigger conditions via UI — no restart required.
🔑
OCP OAuth2 SSO
Single sign-on with OpenShift groups. Three-tier role model: admin, editor, viewer. redirectURIs auto-synced on startup — zero manual OAuth configuration.
🚫
Exclusion Filters
Drop noisy service account traffic before it hits the database. Wildcard actor matching. Reloaded every 30s — no pod restarts. Keeps your database lean and signal clean.
📊
Summary Dashboard
Cluster overview with 24h activity chart, top actors, top resources, verb breakdown, namespace heatmap, and source distribution. Spot anomalies instantly.
📁
CSV Export
Export any filtered view to CSV for compliance reporting, SIEM ingestion, or offline analysis. One click, respects all active filters.
🔍
Field-Level Diffs
With WriteRequestBodies audit policy enabled, see exactly what changed — replica count, image tags, environment variables. Full request body capture.
🛡️
SOC2 / PCI Ready
Full audit trail of human and system actions across all namespaces. Retention policies, role-based access, immutable event log from kube-apiserver.

How it works

Five components, one namespace, zero external dependencies.

01
kube-apiserver
Emits audit events for every API call
02
CLF
OpenShift Logging forwards to collector via HTTP
03
Collector
Normalizes, filters exclusions, stores in PostgreSQL
04
Analyzer
Granite 3.2 scores each event HIGH / MED / LOW
05
UI + Alerter
Live stream, dashboard, alerts to Slack / email
🤖 IBM Granite 3.2 · Ollama · On-cluster

AI that stays
inside your cluster

Every audit event is scored by IBM Granite 3.2 — a deliberate choice for Red Hat environments. The model runs locally via Ollama. No event ever leaves your cluster.

  • Granite 3.2:2b pulled once at deploy time (~1.5GB)
  • Batch scoring: 20 events every 30 seconds
  • Deterministic temperature (0.1) for consistent results
  • Works without GPU — CPU inference on any node
  • Disable with a single Helm flag if not needed

// quick start

One command to deploy

Helm chart handles everything — secrets, RBAC, OAuthClient, groups. No manual configuration required.

Prerequisites
✓
OpenShift 4.x + cluster-admin
Any OCP 4.x cluster with cluster-admin access
✓
Helm 3.x
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
✓
OpenShift Logging Operator
oc apply -f deploy/04b-logging-operator.yaml
⚠
Audit Policy (optional)
Enables field-level diffs. Triggers kube-apiserver rolling restart (~5-10 min, cluster stays available)
bash · the only command you need
# Deploy — all secrets, RBAC, OAuthClient, groups created automatically
$ helm install audit-radar ./Helm/audit-radar
NAME: audit-radar
LAST DEPLOYED: Thu Mar 12 09:36:59 2026
STATUS: deployed
# Then add yourself as admin and open the UI
$ oc adm groups add-users audit-radar-admins your-username
$ oc get route audit-ui -n audit-vision
audit-ui-audit-vision.apps.your-cluster.com ✓
